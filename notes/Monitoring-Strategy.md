# Portfolio Watch Skill — 监控策略设计

> 本文档是 Portfolio Watch Skill 的核心方法论：盯哪些维度、什么算异动、什么是噪音、多信号如何排序。
> 设计目标：对任意未见过的持仓组合生效（可复用），且用户零配置也能得到高质量默认行为。

## 0. 一条总设计原则

**所有阈值都是相对的，不存在硬编码的绝对数字。**
NVDA 跌 3% 是日常，可口可乐跌 3% 是大事。Skill 在 Setup 阶段为每只持仓自动建立"个股画像"（波动率、均量、β、流动性分层），所有异动判定都以该股票自身的统计基线为尺子。这是"对没见过的持仓也生效"的根本机制。

## 1. 盯哪些维度（四层监控模型）

按"投资者为什么会关心"组织，而不是按数据源组织：

### A. 价格行为层（分钟/小时级轮询）
| 信号 | 定义 | 强度 |
|---|---|---|
| 异常涨跌 | 当日涨跌幅相对自身 20 日波动率的 z-score | 核心 |
| 跳空缺口 | 开盘价 vs 前收盘，σ 归一化 | 核心 |
| 成交量异常 | **RVOL（相对成交量）= 当前量 / 同一时段 10–20 日均量**，须按"当日时点"归一化（成交量本身随交易时段起伏）。2x=明显、3x=强、5x+=重大催化（财报/并购）。**注意：5x+ 往往是climactic/衰竭信号，不等于趋势延续，需与价格方向结合判读** | 确认器 |
| 盘前/盘后异动 | 盘外时段显著移动（常由新闻/财报驱动） | 核心 |
| 关键位突破 | 52 周新高/新低、创上市新高等离散里程碑 | 弱信号 |
| 慢趋势 | 连续 N 日下跌、5 日累计跌幅超 2σ（单日不显著但累积显著） | 慢信号 |

### B. 事件层（日历驱动，可预知）
- 财报：前 1-2 天预告提醒 + 发布后结果 vs 预期 + 指引变化
- 分红 / 拆股 / 除权除息
- 大额解禁、股东大会等公司日程
- 宏观日历（FOMC、CPI）——仅当组合对其敏感（高 β 或利率敏感行业）时提醒

### C. 信息层（新闻驱动，不可预知）
- 重大公司新闻：M&A、诉讼、监管调查、高管变动、产品重大事故/召回
- 分析师评级与目标价变动——**聚合处理**（单条微调是噪音，密集同向变动是信号）
- 监管文件：8-K 重大事项、大额内部人交易

### D. 组合层（用户真正的关切，也是界面首屏）
- 组合当日 P&L 异动（相对组合自身历史日波动）
- 回撤：从近期高点回落超过组合波动率的合理范围
- 集中度：单一持仓权重被动漂移超阈值（如涨出来的 40% 仓位）
- 相关性收敛：持仓间相关性突然升高 = 分散化失效，整个组合变成一个 bet
- **风格/因子暴露漂移**：行业、国别、货币（含 ADR/海外持仓的汇率敞口）、以及估值/动量/质量等因子的加权暴露发生显著偏移——机构做组合监控的标准维度，回答"我的组合悄悄变成了什么"
- **流动性分层**：按可变现时间对持仓分层，标记难以快速退出的部分（机构常态化监控项）

**取舍说明**：期权 IV、空头持仓变化等衍生品信号有价值但数据可得性差、解释成本高，V1 不做，留作扩展点。

## 2. 什么算异动（三重检验）

一个信号要成为"真异动"，依次过三关：

1. **统计显著**：|move| ≥ k × σ(20d)。k 分档：≥2.0 进界面高亮，≥2.5 可推送，≥3.5 强制推送。尺子是股票自己的波动率，不是固定百分比。
2. **独立性**：剔除市场分量后仍显著。idio_move = 股票涨跌 − β × 指数涨跌。大盘跌 3% 时持仓跌 3% 不是个股新闻——见噪音规则 #2。
3. **确认/归因**：伴随异常成交量（>2x 均量）或可归因到具体新闻/事件 → 升级；无量、无因的纯价格 blip → 降一档，先观察，若持续再报。

事件层与信息层信号（财报、M&A 等）天然是"真异动"，不需要过统计关，直接按影响面定级。

## 3. 什么是噪音（显式排除清单）

1. 正常波动带内的日内震荡（|z| < 2 且当日内均值回归）。
2. **β 驱动的同步波动**：大盘/板块整体移动时，不为每只持仓单独报警，而是上卷成**一条**组合级 alert（"市场普跌 2.8%，你的组合 β 加权预期跌 3.1%，实际 3.0%，无个股异常"）。这是把 10 条噪音变 1 条信号的关键产品决策。
3. **同一事件的重复报道**：按 story 聚类去重，同一 ticker 同一主题 24h 冷却。
4. **低流动性时段假动作**：盘前低成交量的跳动不立即报，等开盘后确认。
5. **已报告行情的延续**：昨天报过"跌 3σ"，今天继续阴跌不再报——除非严重度升级到新档位（ratchet 机制：只升级、不重复）。
6. 分析师单条目标价小幅调整：进界面聚合视图，不单独成信号。

## 4. 多信号同时出现怎么排序

### 评分模型
```
priority = severity × portfolio_impact × novelty
```
- **severity**：异动幅度（σ 单位）或事件级别（财报爆雷 > 评级下调）
- **portfolio_impact**：|move%| × 持仓权重。核心直觉：**1% 仓位的 5σ 异动，可能不如 40% 仓位的 2.5σ 异动重要**——排序按"对用户钱包的影响"，不按"新闻的响度"
- **novelty**：全新信息 = 1.0；已知事件的后续 = 0.5；已报告行情的延续 = 0（被 ratchet 拦截）

### 三档处置
| 档 | 定义 | 处置 |
|---|---|---|
| P0 | 财报后大幅跳空、重大公司事件、组合回撤破位、≥3σ 有量有因 | 立即单条推送，带 deep link 直达界面对应卡片 |
| P1 | 2–2.5σ 确认异动、评级密集变动、慢信号累积触发 | 合并进摘要推送（每天最多早/晚两个 digest 窗口） |
| P2 | 接近阈值、里程碑类、FYI | 只进界面，不推送 |

### 并发合并规则（防轰炸）
1. **同 ticker 多信号 → 融合成一条**：价格异动 + 放量 + 新闻同时命中，不发三条，发一条完整叙事（信号互为确认，优先级取最高档）。
2. **多 ticker 同因 → 上卷成组合级一条**：市场/板块驱动的集体移动合并汇报。
3. **推送预算**：P0 每天默认 ≤ 4 条；超预算按分数截断，落选的降级进 digest。宁可漏报一条中等信号，不可让用户把通知调成静音——alert 的信任是这个产品的核心资产。
4. **冷却**：同 ticker 同维度 4 小时冷却，严重度升级可穿透。

## 5. 可复用性机制（Setup 阶段）

用户只需给出持仓（权重可选，缺省等权）。Skill 自动完成：
1. 逐 ticker 拉取历史，计算 20/60 日波动率、均量、对基准的 β；
2. 识别资产类型（个股/ETF/ADR）与流动性分层，微调确认规则（ETF 免个股新闻监控，低流动性股提高量能确认门槛）；
3. 拉取财报/分红日历，注册事件监听；
4. 所有阈值由画像导出——换任何组合，同一套规则自动适配。

灵敏度提供三档预设：**安静**（只报 P0）/ **标准**（默认）/ **敏感**（P1 也即时推）。用户用自然语言即可切换（"太吵了" → 降档）。

## 附：策略的研究依据

这套设计不是拍脑袋，几条核心决策都能对到成熟方法论，且在两个独立领域（量化金融 + 运维/安全告警工程）交叉验证：

**异动检测（量化金融）**
- z-score / 滚动统计是异动检测的标准基线；主流做法强调**组合多指标、避免单指标告警**，与本策略"三重检验"一致。
- 成交量用 **RVOL（相对成交量）**并按时点归一化：2x 明显、3x 强、5x+ 重大催化（财报/并购）；但 5x+ 常是衰竭而非延续——精化了 V1 里"2x 均量"的粗略写法。
- **β 分解出个股独立收益（idiosyncratic）**是过滤"市场噪音 vs 个股新闻"的标准手段：大盘驱动的同步移动不构成个股信号——直接支撑噪音规则 #2 的 β 上卷。

**告警排序与去噪（运维/安全告警工程，SRE/SOC 成熟学科，可直接迁移）**
- **按严重度路由**：critical → 即时推送/呼叫，低优 → 仅进看板——即本策略 P0/P1/P2 分档。
- **去重 = 状态未实质变化时更新原告警，而非发新通知**——正是本策略的 ratchet（只升级不重复）机制，跨领域独立印证。
- **动态基线 / 自适应阈值**（按历史与时段季节性定阈）是抗告警疲劳的核心——即本策略"所有阈值相对化"总原则。

**监控维度（机构组合风控）**
- 机构标准维度含：持仓与权重、行业/国别/货币敞口、流动性分层、因子（估值/动量/质量）暴露——据此补入组合层的"因子漂移"与"流动性分层"。
- 散户告警产品实际触发项（价格%、52 周高低、财报、分红、评级变动、放量、CPI 等宏观）与四层模型覆盖面吻合。
